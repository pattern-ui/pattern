---
title: 'Tooltip component error'
error: 'Tooltip component children should be an element or a component that accepts ref, fragments, strings, numbers and other primitive values are not supported'
---

## Error source

[Tooltip](/core/tooltip/) was used with invalid children:

```tsx
import { Tooltip } from '@pattern/core';

function DemoWithError() {
  return (
    <>
      <Tooltip label="Throws">Raw string, NOT OK – will throw error</Tooltip>
      <Tooltip label="Throws">{2}</Tooltip>

      <Tooltip label="Throws">
        <>Fragment, NOT OK, will throw error</>
      </Tooltip>

      <Tooltip label="Throws">
        <div>More that one node</div>
        <div>NOT OK, will throw error</div>
      </Tooltip>
    </>
  );
}
```

## How to fix

Tooltip children should be either an element or component that supports ref:

```tsx
import { Tooltip, Badge } from '@pattern/core';

function Demo() {
  return (
    <>
      <Tooltip label="OK">
        <button>Native button – ok</button>
      </Tooltip>

      <Tooltip label="OK">
        <Badge>Pattern component – ok</Badge>
      </Tooltip>
    </>
  );
}
```
