## Pattern UI

[Pattern UI](https://ui.pattern-ui.design/) is a new project with set of more than 120 responsive components built with Pattern.
All components support dark/light color scheme and Pattern theme customizations.
Pattern UI is free for everyone.

- [Pattern UI website](https://ui.pattern-ui.design/)
- [Pattern UI source code](https://github.com/pattern-ui/ui.pattern-ui.design)

## Breaking changes

- `@pattern-ui/core` package no longer exports `CONTAINER_SIZES` variable
- [Prism](https://pattern-ui.design/others/prism/) now trims code by default, to disable that set `<Prism trim={false} />`
- `useForm` hook exported from `@pattern-ui/hooks` package is now deprecated, it will be removed in one of the next releases.
- [@pattern-ui/eslint-config](https://www.npmjs.com/package/@pattern-ui/eslint-config) was migrated to [eslint-config-pattern](https://www.npmjs.com/package/eslint-config-pattern)

## Renamed props

Some props were renamed to make Pattern components more consistent:

- [Dialog](https://pattern-ui.design/core/dialog/), [Card](https://pattern-ui.design/core/paper/), [Paper](https://pattern-ui.design/core/paper/), [Header and Navbar](https://pattern-ui.design/core/app-shell/)
  - `padding` → `p`
- [Container](https://pattern-ui.design/core/container/)
  - `padding` → `px`
- [Radio](https://pattern-ui.design/core/radio-group/)
  - `children` → `label`
- [RadioGroup](https://pattern-ui.design/core/radio-group/):
  - `variant` → `orientation`
- [Tooltip](https://pattern-ui.design/core/tooltip):
  - `delay` → `closeDelay`
- [Popover](https://pattern-ui.design/core/popover/):
  - `noClickOutside` → `closeOnClickOutside`
  - `noFocusTrap` → `trapFocus`
  - `noEscape` → `closeOnEscape`
- [Modal](https://pattern-ui.design/core/modal/):
  - `noFocusTrap` → `trapFocus`
  - `hideCloseButton` → `withCloseButton`
- [Drawer](https://pattern-ui.design/core/drawer/):
  - `noFocusTrap` → `trapFocus`
  - `hideCloseButton` → `withCloseButton`
  - `noOverlay` → `withOverlay`
  - `noCloseOnEscape` → `closeOnEscape`
  - `noCloseOnClickOutside` → `closeOnClickOutside`
  - `noScrollLock` → `lockScroll`

## Default props on PatternProvider

Most of components now support default props on PatternProvider:

```tsx
import { PatternProvider, Button } from '@pattern-ui/core';

function App() {
  return (
    <PatternProvider
      defaultProps={{
        Button: { color: 'red' },
        Badge: { size: 'xl', radius: 0 },
        // ... default props for other components
      }}
    >
      {/* By default, Button will have red color */}
      <Button>Red button</Button>

      {/* Default color can be overwritten by props */}
      <Button color="blue">Blue button</Button>

      {/* By default, Badge will have xl size and 0 radius */}
      <Badge>Badge</Badge>
    </PatternProvider>
  );
}
```

## Styles params on PatternProvider

You can now get component styles params in [PatternProvider](https://pattern-ui.design/theming/pattern-provider/) `styles` prop.
Each component that has [Styles API](https://pattern-ui.design/theming/styles-api/) now exports type that can be assigned to params
`ComponentNameStylesParams`, for example, `ButtonStylesParams`. This feature can be used to add more complex context styles that were not possible before:

```tsx
import { PatternProvider, ButtonStylesParams } from '@pattern-ui/core';

function Demo() {
  return (
    <PatternProvider
      styles={{
        Button: (theme, params: ButtonStylesParams) => ({
          root: {
            // use params to calculate dynamic styles
            color: theme.colors[params.color || theme.primaryColor][1],
            padding: params.size === 'lg' ? '15px 45px' : undefined,
          },
        }),
      }}
    />
  );
}
```

## Nested PatternProvider inheritance

Nested PatternProviders will now inherit `theme` override, `emotionOptions`, `defaultProps` and `styles` from
parent provider if `inherit` prop is set:

```tsx
import { PatternProvider, Button } from '@pattern-ui/core';

function App() {
  return (
    // Parent PatternProvider
    <PatternProvider
      theme={{ colorScheme: 'dark' }}
      styles={{ Button: { root: { fontWeight: 400 } } }}
      defaultProps={{ Badge: { variant: 'outline' } }}
      emotionOptions={{ key: 'custom-cache' }}
    >
      <Button>Affected only by parent provider</Button>

      {/*
        Child PatternProvider, inherits theme, emotionOptions, defaultProps and styles
        from parent PatternProvider. Other properties specified on child provider will override parent props.
        For example, theme override will be: { colorScheme: 'dark', primaryColor: 'red' }
      */}
      <PatternProvider theme={{ primaryColor: 'red' }} inherit>
        <Button>Affected only by child provider</Button>
      </PatternProvider>
    </PatternProvider>
  );
}
```

## Default radius and focus ring configuration on theme

Pattern theme now includes two new properties: `defaultRadius` and `focusRing`.

`defaultRadius` property is used to set border-radius on most components by default:

```tsx
import { PatternProvider, Button } from '@pattern-ui/core';

function App() {
  return (
    <PatternProvider theme={{ defaultRadius: 0 }}>
      <Button>Zero radius button</Button>
      <Button radius="xl">xl radius button</Button>
    </PatternProvider>
  );
}
```

With `focusRing` property you can configure how focus ring is displayed:

- `auto` – display focus ring only when user navigates with keyboard (default)
- `always` – display focus ring when user navigates with keyboard and mouse
- `never` – focus ring is always hidden (not recommended)

```tsx
import { PatternProvider, Button } from '@pattern-ui/core';

function App() {
  return (
    <PatternProvider theme={{ focusRing: 'always' }}>
      <Button>Focus ring will be displayed when user clicks the button</Button>
    </PatternProvider>
  );
}
```

## Padding props

All components now support setting following props to control padding:

- `p` – sets `padding` property on root element
- `py` – sets `padding-top` and `padding-bottom` properties on root element
- `px` – sets `padding-right` and `padding-left` properties on root element
- `pt` – sets `padding-top` property on root element
- `pb` – sets `padding-bottom` property on root element
- `pl` – sets `padding-left` property on root element
- `pr` – sets `padding-right` property on root element

```tsx
import { Paper, Container } from '@pattern-ui/core';

function Demo() {
  return (
    <>
      <Paper p={20} />
      <Container px="xl" />
    </>
  );
}
```

## createStyles references changes

[createStyles](https://pattern-ui.design/theming/create-styles/) function now works differently with references.
Instead of generating unique identifiers with each `getRef` call it returns the same value for given argument:

```tsx
// before 4.0.0
// getRef('icon') !== getRef('icon')
createStyles((theme, params, getRef) => {
  const icon = getRef('icon');
  return {
    icon: { ref: icon },
    parent: {
      [`& .${icon}`]: { color: 'red' },
    },
  };
});

// in 4.0.0
// getRef('icon') === getRef('icon')
createStyles((theme, params, getRef) => ({
  icon: { ref: getRef('icon') },
  parent: {
    [`& .${getRef('icon')}`]: { color: 'red' },
  },
}));
```

## Other changes

- New [use-disclosure](https://pattern-ui.design/hooks/use-disclosure/) hook
- [Text](https://pattern-ui.design/core/text/) component now inherits color from parent element
- [Input](https://pattern-ui.design/core/input/) component now handles `required` prop differently to support native browser validation
- [RichTextEditor](https://pattern-ui.design/others/rte/) now supports providing extra quill modules via `modules` prop
- [DateRangePicker](https://pattern-ui.design/dates/date-picker/) now displays first selected date
- [RangeCalendar](https://pattern-ui.design/dates/calendar/) now calls onChange with first selected date
- [SegmentedControl](https://pattern-ui.design/core/segmented-control/) component now supports vertical orientation and disabled state
- [ThemeIcon](https://pattern-ui.design/core/theme-icon/) component now supports outline variant
- [Text](https://pattern-ui.design/core/text/), [Anchor](https://pattern-ui.design/core/anchor/) and [Highlight](https://pattern-ui.design/core/highlight/) components now support `underline` prop
- [Container](https://pattern-ui.design/core/container/) component now supports sizes configuration with default props on PatternProvider
- All components now support negative margins for theme values with margin props (`m`, `mx`, `my`, `mt`, etc.): `<Button mx="-xl">Negative margins</Button>`
- `onCancel` callback in [@pattern-ui/modals](https://pattern-ui.design/others/modals/) is now also called when confirm modal is closed without confirmation
- New [OptionalPortal](https://pattern-ui.design/core/portal/#optionalportal-component) component can be used to switch between rendering within Portal or as a regular child with `withinPortal` prop
- [Modal](https://pattern-ui.design/core/modal/) and [Drawer](https://pattern-ui.design/core/drawer/) can now be used without [Portal](https://pattern-ui.design/core/portal/)
- [Title](https://pattern-ui.design/core/title/) component now inherits color from parent element
