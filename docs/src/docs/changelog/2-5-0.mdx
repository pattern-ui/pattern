## UMD builds deprecation

All `@pattern/` packages will no longer default to umd builds via `browser` field in `package.json`.
This change will simplify tree shacking for Webpack and should not affect your Next.js, Gatsby, Vite or CRA applications.

## react-jss peer dependency removal

All `@pattern/` packages no longer require react-jss installation as peer dependency – it will be installed automatically as @pattern/core dependency.
Instead of direct react-jss usage @pattern/core now exports [createStyles](https://pattern-ui.design/theming/react-jss/) function
and a [set of components](https://pattern-ui.design/theming/next/) to implement server side rendering.

### createStyles function

New `createStyles` function lets you create styles with Pattern theme:

```tsx
import { createStyles } from '@pattern/core';

const useStyles = createStyles((theme) => ({
  button: {
    backgroundColor: theme.colors.blue[6],
    color: theme.white,
    padding: `${theme.spacing.md}px ${theme.spacing.lg}px`,
  },
}));

function Demo() {
  const classes = useStyles();
  return <button className={classes.button}>My button</button>;
}
```

Note that it will only work if your application is wrapped with PatternProvider.

### Builtin server side rendering

@pattern/core package now comes with components to simplify server side rendering setup with Next.js and any other environment.
Checkout [Next.js guide](https://pattern-ui.design/theming/next/) to learn more.

## New hooks

- [use-pagination](https://pattern-ui.design/hooks/use-pagination/) – manage pagination state
- [use-focus-return](https://pattern-ui.design/hooks/use-focus-return/) – capture last focused element on the page and return focus to it once condition is met

## New features

- [Button](https://pattern-ui.design/core/button/#white-variant) component now supports new `white` variant
- [Text](https://pattern-ui.design/core/text/#inherit-styles) now supports font styles inheritance from parent element with `inherit` prop
- [Paper](https://pattern-ui.design/core/paper/) component now supports `withBorder` prop to add 1px border
- [ActionIcon](https://pattern-ui.design/core/action-icon/) component now supports `loading` state (same as in [Button](https://pattern-ui.design/core/button/) component)
- [SegmentedControl](https://pattern-ui.design/core/segmented-control/) component now supports setting data as an array of strings `<SegmentedControl data={['React', 'Angular', 'Svelte', 'Vue']} />`
- [NumberInput](https://pattern-ui.design/core/number-input/#decimal-steps) component now supports decimal steps and precision
